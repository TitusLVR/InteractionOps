# Rust crates for InteractionOps

This is a Cargo workspace. Each subdirectory is an independent crate that
compiles to a `.pyd` (Python extension) and is copied into
[`../operators/`](../operators/) at the addon root, where the Python side
loads it via `import <crate_name>`.

## Crates

| Crate | Purpose | Target binary |
|---|---|---|
| [`mesh_uv_shortest_mark_lib`](mesh_uv_shortest_mark_lib/) | Graph, A*, Dijkstra, nearest-vertex for the Shortest Path Mark operator | `operators/mesh_uv_shortest_mark_lib.pyd` |

## Adding a new crate

1. `cargo new --lib <name>` inside this folder
2. Add `<name>` to the `members` list in [Cargo.toml](Cargo.toml)
3. Set `crate-type = ["cdylib"]` and depend on `pyo3` with `features = ["extension-module", "abi3-py311"]`
4. Expose a `#[pymodule]` named `<name>` from `src/lib.rs`
5. Build and deploy (see below). The Python side imports it by name — either
   `from . import <name>` (package context) or `import <name>` (when the
   addon's `operators/` is on `sys.path`).

## Building

Requires:

- Rust stable toolchain (`rustup`, `cargo` on PATH)
- A Python 3.11 install with dev libs (`libs/python3.lib` + `libs/python311.lib`)
  pointed at via `PYO3_PYTHON`
- MSVC Build Tools / VS 2022 installed (Rust auto-detects)

Build the whole workspace:

```powershell
$env:PYO3_PYTHON = "C:\Users\<you>\AppData\Local\Programs\Python\Python311\python.exe"
cd rust
cargo build --release
```

Then copy each produced `.dll` to `operators/<crate>.pyd`:

```powershell
Copy-Item target\release\mesh_uv_shortest_mark_lib.dll ..\operators\mesh_uv_shortest_mark_lib.pyd
```

On Windows the `.pyd` is file-locked while Blender has the addon loaded — if
the copy fails with "Device or resource busy," close Blender first, or use a
move-then-copy trick:

```bash
mv ../operators/mesh_uv_shortest_mark_lib.pyd ../operators/mesh_uv_shortest_mark_lib.pyd.old
cp target/release/mesh_uv_shortest_mark_lib.dll ../operators/mesh_uv_shortest_mark_lib.pyd
rm ../operators/mesh_uv_shortest_mark_lib.pyd.old
```

## Conventions

- **ABI**: `abi3-py311` — the same binary works with Blender's bundled
  Python 3.11 and any future 3.11+ release without a rebuild
- **Release profile**: `lto = "fat"`, `codegen-units = 1`, `panic = "abort"` —
  set at the workspace root, so every crate benefits
- **Python fallback**: every crate loaded from `operators/` should be wrapped
  with a `try / except ImportError` on the Python side so the addon still
  works when the `.pyd` is absent

## Not committed

- `target/` — build artifacts (large; regenerated by `cargo build`)
- `Cargo.lock` — regenerated by `cargo build`; fine to delete or keep
